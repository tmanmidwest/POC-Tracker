💡 Prefer to use Claude directly? (often more thorough)

This importer makes a single AI pass over the document. If you've connected POC Tracker as an MCP server in Claude (Desktop, claude.ai, or an IDE), you can instead attach the document there and ask Claude to add the use cases — for example: “Read this requirements doc and add any new use cases to the {{ project.display_name }} POC.”

That route is usually more thorough because Claude works as an agent: it can read the whole document, call get_project to see what's already there and de-duplicate, ask you clarifying questions, and add the use cases directly via add_custom_use_cases — all in one conversation. The trade-off is that it needs the MCP set up and a Claude client; this in-app importer needs neither. An admin configures the connection under Settings → MCP.
