How the MCP server picks this up

The MCP server reads its token in this order:

  1. POCT_MCP_API_KEY env var, if set (a fixed override — leave it unset to use this UI-managed token).
  2. POCT_MCP_API_KEY_FILE env var, if set.
  3. Otherwise the file above on the shared data volume.

For auto-rotation to work, the MCP server must share this app's data directory (same host, or the same Docker volume). A remote MCP host can't read the file — set POCT_MCP_API_KEY there instead.
