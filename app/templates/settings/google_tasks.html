This is a two-way, per-user integration. You provide one Google OAuth client here; each user then connects their own Google account from the Tasks page. Set it up in the Google Cloud console: create an OAuth client (type “Web application”), enable the Google Tasks API, add the .../auth/tasks scope on the consent screen, and register this redirect URI:

Copy this into your Google OAuth client’s “Authorized redirect URIs”.
Consent screen: use “External”

Because users connect their own personal / non-org Google accounts, set the OAuth consent screen “User type” to External (Internal only works if everyone shares one Google Workspace org). Then mind the publishing status:

  • Testing (default): only Google accounts you add under Test users can connect (max 100), and their access expires about every 7 days — they’ll be prompted to reconnect here (that’s expected).
  • In production (published): removes those limits. Since .../auth/tasks is a “sensitive” scope, Google may require app verification before publishing. Once verified, connections are long-lived.
